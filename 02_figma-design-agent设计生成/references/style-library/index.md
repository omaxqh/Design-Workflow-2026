# 风格素材库索引

本文件用于索引风格参考图。风格素材库只作为 image 2 生成阶段的美学校准参考，用于提升视觉质感、模块层级、留白节奏、色彩氛围和装饰克制度。

风格参考不得覆盖上游设计需求、`design-schema.md` 或 Figma 原页面结构。

## 使用规则

每次生成 image 2 前，根据任务选择 1-3 张最相关参考图。

目录名只表示素材分类或来源场景，不是检索边界。不要只按目标页面目录查找素材。先从需求中提取业务主题、目标模块、改动意图和可迁移视觉模式，再跨目录查找最相关参考图。

## 选择优先级

优先按以下顺序选择参考图：

1. 目标模块
2. 业务主题
3. 改动意图
4. 可迁移的视觉模式
5. 页面类型
6. 业务氛围
7. 色彩方向
8. 模块密度
9. 组件质感

检索方式：

- 提取业务主题：如 AI、会员、栏目、价格、权益、营销、服务、内容、工具、任务、支付等。
- 提取目标模块：如推荐模块、栏目入口、权益卡片、价格说明、服务保障、内容频道等。
- 提取改动意图：如新增入口、强化转化、提升说明效率、增加个性化、聚合信息等。
- 提取可迁移视觉模式：如卡片组、横滑券卡、宫格入口、对比表、任务列表、推荐流、浮层、信息看板等。
- 从全部目录中找候选参考，不以目标页面目录为限制。
- 只保留 1-3 张最能帮助主视觉稳定的参考图。

`可迁移到` 是建议，不是限制。如果某张图的主题标签、适用模块或视觉模式与当前任务高度相关，即使 `可迁移到` 没写目标页面，也可以选用。

## 只参考

- 视觉质感
- 模块层级
- 留白节奏
- 色彩氛围
- 卡片质感
- 装饰克制度
- 图标或插画抽象风格
- 与目标模块相关的可迁移视觉模式

## 禁止照搬

- 页面结构
- 业务内容
- 文案
- 品牌元素
- 与 `design-schema.md` 冲突的组件样式

如果参考图与 `design-schema.md` 冲突，以 `design-schema.md` 为准。

## 索引格式

每张参考图按以下格式登记：

```markdown

禁止照搬：
备注：
```
