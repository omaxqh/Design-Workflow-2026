# Fliggy Figma 产品级设计规范

> 来源：`fliggy-design-gui v0.3.4` 中的产品设计基础、组件分类、页面索引与视觉规则。
>
> 目标：服务 `figma-design-agent` 的 **Figma 可编辑设计稿还原**。本文件只导入产品级设计规范，不导入 HTML 还原、DOM/CSS 实装、单文件 HTML 交付、`example.html` 拷贝等执行规则。

## 0. 边界

本 agent 的最终交付是 Figma 设计稿，不是 HTML 页面。

必须保留：

- 飞猪 App 移动端产品视觉基准。
- 色彩、字体、字号、行高、圆角、间距、栅格、阴影层级。
- 平台组件的产品语义、状态、尺寸和使用场景。
- 页面类型、业务域、信息层级和模块节奏。
- 图片、图标、品牌氛围和无障碍语义对设计稿的约束。

必须排除：

- 单文件 HTML、内联 `<style>`、`viewport`、`@font-face` 写法。
- DOM、CSS class、BEM 命名、`var(--*)`、`example.html` 整页拷贝。
- `spec.md + example.html` 作为代码真相源的交付规则。
- 浏览器预览、HTML 设计审查、真实数据替换脚本等实现流程。

Figma 中需要把规范落为：frame、component、variant、text style、color variable、effect style、auto layout 和可编辑图层。

## 1. 设计基准

- **目标平台**：飞猪 App 移动端页面。
- **设计画布**：默认使用 `750 × 1624px` 的移动端画板，规范标注按 `@2x px` 理解。
- **页面宽度**：常规页面主宽 `750px`；通栏模块可满宽；内容卡片常见宽度为 `714px`（左右 18px 边距）或按页面规范约定。
- **布局方向**：以纵向滚动页面为主；固定导航、底栏、浮层等按 Figma 图层结构表达。
- **还原要求**：最终稿必须可编辑、结构化，禁止把整张方案图或整页截图作为最终设计。

## 2. Figma Token 来源

优先使用项目内 token 文件：

- `references/design-system/agent-tokens.json`

使用原则：

- Figma 颜色优先匹配 token 中已有变量或样式。
- 若产品规范中的语义色与 Figma token 命名不同，按语义匹配，不按字符串硬套。
- 新增局部颜色、渐变或效果时，必须说明业务用途；能映射到已有 token 的不新增。
- 方案图里的非规范色只作为视觉意图参考，落 Figma 时必须转成最接近的规范色。

## 3. 字体与排版

### 3.1 字体

- 默认中文：`PingFang SC`，备用 `SimHei` 或系统中文无衬线。
- 数字 / 货币：优先使用飞猪数字字体语义；若当前 Figma 文件无对应字体，使用项目已有数字样式或最接近样式。
- 品牌汉字：用于品牌向标题、活动氛围字、Banner 主标题等；需要体现倾斜、较强识别感和中等字重。

### 3.2 字号阶梯

| 层级 | 字号 | 字重 | 行高 |
|------|------|------|------|
| H1 | 48px | 500 | 140% |
| H2 | 42px | 500 | 140% |
| H3 | 36px | 500 | 140% |
| H4 | 30px | 500 | 140% |
| Paragraph | 30px | 400 | 170% |
| Body 1 | 28px | 400 / 500 | 140% |
| Body 2 | 26px | 400 / 500 | 140% |
| Body 3 | 24px | 400 / 500 | 140% |
| Body 4 | 22px | 400 / 500 | 140% |
| Caption | 20px | 400 / 500 | 140% |
| Caption 1 | 18px | 400 | 140% |

### 3.3 字重规则

- 标题、选中项、按钮、强调信息使用 `500`。
- 正文、说明、未选中项使用 `400`。
- 价格数字可用 `600 / 700`，并允许视觉底对齐和更紧凑行高。
- 同一模块内不要混用过多字号；优先用字重、颜色和间距建立层级。

## 4. 色彩

### 4.1 核心语义色

| 语义 | 推荐值 | 用途 |
|------|------|------|
| 深色字 / 深色图标 | `#0F131A` | 标题、正文、强层级文字 |
| 次级文字 | `#5C5F66` | 副标题、标签、次按钮文字 |
| 弱说明 | `#919499` | 弱说明、副文案、明细 |
| 禁用 / 弱边框 | `#D2D4D9` | 失效文案、禁用边框 |
| 页面灰底 | `#F2F3F5` | 页面底层背景、分割线、次级底 |
| 浅灰辅底 | `#F7F8FA` | 浅控件底、卡片内色块 |
| 白色卡片 | `#FFFFFF` | 卡片主背景、面板背景 |
| 品牌黄 | `#FFE033` | 主按钮、选中态、核心强调 |
| 品牌浅黄 | `#FFFCEB` | 轻选中底 |
| 品牌辅助黄 | `#FFF7CC` | 复选选中、区间选中、浅品牌底 |
| 靛蓝 | `#6666FF` | 链接、焦点、次级选中 |
| 靛蓝浅底 | `#EBEBFF` | 靛蓝浅标签、时间轴连线 |
| 警告红 | `#FF3333` | 删除、警告 |
| 支付橙红 | `#FF5533` | 价格、支付、优惠 |
| 次级转化橙 | `#FF8C1A` | 加入购物车等次级转化按钮 |
| 大促红 | `#FF0036` | 大促角标、小面积营销标识 |
| 大促紫 | `#9537FD` | 预售角标、小面积营销标识 |
| 正向绿 | `#149AA8` | 正向反馈、提示 |
| 会员浅底 | `#FEF5EC` | 会员 / 保障浅底 |
| 会员深字 | `#5D3521` | 会员 / 保障深字 |

### 4.2 使用规则

- 品牌黄用于核心选择和主操作，不作为大面积页面背景。
- `#FF5533` 用于价格和支付强转化，避免与警告红混用。
- 大促红 / 大促紫只用于角标、小标签、局部活动识别，禁止整屏铺色。
- 页面灰底用于底层；卡片底优先白色；卡片内部弱分区用浅灰辅底。
- 文案可读性优先，摄影图或渐变上的文字必须保证对比度。

## 5. 圆角

| Token / 规则 | 值 | 用途 |
|------|------|------|
| Radius L | 24px | 对话框、浮层、通栏卡片、大模块 |
| Radius M | 12px | 非通栏卡片、常规内容卡 |
| Radius S | 6px | 小标签、小控件 |
| 标签小圆角 | 4px | 特定小标签、活动角标 |
| 胶囊 | 高度的 1/2 | 主按钮、胶囊筛选、浮动操作 |

页面级规范可覆盖产品级圆角。例如酒店下单页模块整体常用 12px，列表和详情新增模块常用 24px。

## 6. 栅格、间距与版心

- 基础单位：6px。
- 常规页面水平边距：18px。
- 首页或重点内容流可使用 30px 水平边距，以页面级规范为准。
- 模块内边距常用：18px、24px。
- 模块内元素间距常用：6px、12px、18px、24px。
- 模块之间垂直间距常用：18px、24px、30px。
- 主内容应使用纵向 auto layout 或清晰分组；模块高度由内容驱动。
- 固定头部、底栏、浮层与内容区需要有明确避让关系，不遮挡主要信息。

## 7. 阴影与层级

- 默认少用阴影，优先用背景色差、分割线、留白和圆角表达层级。
- 确需强调浮层时，使用低透明、低扩散的单层阴影，同类组件保持一致。
- 禁止大模糊、深黑、多层堆叠的通用 AI 阴影。
- 文字叠在图片或渐变上时，可用克制阴影增强可读性，但不要出现厚描边效果。

## 8. 图片与图标

### 8.1 图片

- 含真实配图的页面应使用飞猪业务语境素材：酒店、交通、度假、目的地、民宿、品牌、机票等。
- 同一页面多图不要重复使用同一素材。
- 图片必须体现真实业务内容，不要用纯色块冒充成品配图。
- Figma 中图片应作为独立可替换填充或素材图层，不要合成进整页截图。

### 8.2 图标

- 语义图标优先复用当前 Figma 文件、设计系统或既有页面中的图标组件。
- 找不到组件时，可参考 `fliggy-design-gui` 的图标语义库，但落地到 Figma 时应转成可编辑矢量或组件实例。
- 禁止用 Emoji 或近似图形替代产品图标。
- 新增自绘图标需在交付说明中标注“无现成图标，已自绘”。

## 9. 平台组件语义

Figma 还原时优先复用已有设计系统组件；没有组件实例时，按下列产品语义还原为可编辑结构。

### 9.1 小组件

- 信息气泡：卡片或按钮上的小提示，常用于热销、新品等。
- 小红点 / 数字角标：未读、通知、库存提示。
- Switch：开关设置项。
- 单选 / 复选：选择状态必须清晰区分默认、选中、禁用。
- 数字选择器：数量步进，按钮区域需可触达。
- 筛选标签：单选可用黄底，复选可用角标黄底。
- 轮播条：Banner / 卡片流分页。
- 长按菜单：带箭头的 popover 菜单。
- 时间轴：售后、订单阶段、进度说明。
- 价格：整数放大、支付色、原价删除线、会员价组合。
- 双列推荐：信息流卡片，注意图片、标题、位置、价格、标签层级。

### 9.2 按钮

- 常用按钮：主、次、危险、浅、白、描边等状态。
- 浮动按钮：固定角落操作，可纯图标或图标加短文案。
- 展开 / 收起：胶囊按钮加方向图标。

### 9.3 标签

- 心智类标签：主标 + 描述组合，强调理由或权益。
- 服务类标签：服务承诺、保障、政策信息。
- 大促类标签：双 11、预售、限时等小面积活动识别。
- 优惠类标签：券、红包、折扣，通常和价格 / 结算相关。

### 9.4 导航

- 顶部导航：返回、标题、右侧操作，区分浅色、透明、品牌氛围场景。
- 底部 Tabbar：一级导航、二级底栏、预订底栏。
- Tabs：顶部分类、均分或横滑；选中态要有明确指示。

### 9.5 反馈

- Dialog：遮罩、圆角卡、主次按钮。
- Sheet：底出浮层，内容区可独立滚动。
- Notice Bar：小黄条或通知条。
- Toast：短时状态提示。
- Empty State：插图、标题、描述和主次按钮。

### 9.6 选择

- 时间选择：日历、区间、选中、不可选状态。
- 目的地选择：搜索、Tab、分组城市、索引。
- 优惠券选择：券卡、领取 / 使用 / 勾选 / 失效状态。

### 9.7 展示

- Banner：横向卡片，常包含底图、蒙版、标题、副标题、行动入口和轮播指示。

## 10. 页面类型与信息架构

### 10.1 底栏 Tab

- `tabs.home`：首页 / 推荐。
- `tabs.member`：会员 Tab。
- `tabs.messages`：消息 Tab。
- `tabs.trip`：行程 Tab。
- `tabs.mine`：我的 Tab。

### 10.2 垂直行业

- `vertical.flight.*`：机票首页、列表、OTA、下单、订单。
- `vertical.train.*`：火车票首页、列表、OTA、下单、订单。
- `vertical.hotel.*`：酒店首页、列表、详情、下单、订单。
- `vertical.vacation.*`：门票玩乐、旅行搜索、商详、下单、线路详情、订单。
- `vertical.car.*`：租车首页、列表、OTA、下单、订单。

### 10.3 搜索

- `search.search`：搜索入口。
- `search.search_sug`：搜索联想。
- `search.search_result`：搜索结果。

### 10.4 页面选择规则

- 新增模块：先识别所在页面类型，再读页面级规范和目标 Figma 上下文。
- 修改已有页面：以当前 Figma 页面结构为准，最小范围调整。
- 新增页面：使用本产品级规范确定基础视觉，再参考最接近的页面类型建立信息架构。

## 11. 典型页面约束

### 11.1 首页 / 推荐

- 固定头部、搜索、底栏和可滚内容之间需要层级清晰。
- 主内容模块建议使用纵向流式布局，避免大面积空白。
- 多功能频道、推荐信息流等模块应保持高密度但可读。
- 首页主内容模块常用 30px 左右边距。

### 11.2 酒店首页

- 根页面常用 750 宽纵向布局，模块间距约 18px。
- Banner 承载头图与氛围，不应塞入过多正文信息。
- 酒店频道底栏固定吸底时，内容区需预留安全空间。
- 新增模块少用阴影，优先用分割线、背景层级和卡片圆角。

### 11.3 商品详情 / 线路详情

- 头图、状态栏、导航、Tab、套餐 / 日历 / 购买区需要稳定层级。
- 底操作栏固定时，页面内容必须预留底部空间。
- 内容白卡常用 714px 宽、左右 18px 边距。
- 不要把商详泛化成另一套无关 IA，除非用户明确要求重做信息架构。

## 12. 交互状态与无障碍

Figma 稿需要表达关键状态：

- 默认、悬停 / 按下、选中、禁用、加载、错误、空态。
- 可点击图标需要有明确热区和语义命名。
- 当前导航项、当前 Tab、当前筛选项需要明确选中态。
- 表单、选择器、弹窗、Toast、Sheet 需要体现用户路径和状态反馈。
- 文字和背景对比必须保证可读。

## 13. 禁止事项

- 禁止混用不同设计体系的同名 token。
- 禁止跳过产品级规范直接凭印象画飞猪页面。
- 禁止未查页面级规范就自造模块节奏。
- 禁止用说明性灰块替代真实模块。
- 禁止把整页截图、方案图或 HTML 截图当作最终 Figma 设计稿。
- 禁止使用与业务无关的外部占位图。
- 禁止大面积高饱和铺色、过重阴影、无依据渐变。
- 禁止把 HTML 交付规则迁移到 Figma 交付规则中。

## 14. Figma 落地检查

交付前检查：

- 是否使用 Figma 可编辑结构，而不是整图粘贴。
- 是否优先复用了已有 component、variable、text style。
- 颜色、字号、字重、圆角、间距是否符合本规范和页面级规范。
- 内容是否没有裁切、重叠、溢出。
- 图片是否业务相关且可替换。
- 图标是否语义准确且可编辑或组件化。
- 模块高度是否由内容驱动。
- 新增模块是否与相邻模块节奏一致。
- 是否没有把 HTML / CSS / DOM 说明写进画布。
