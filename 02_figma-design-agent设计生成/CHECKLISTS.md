# 检查清单

## 路由检查

执行前确认：

- 多页面需求已拆成页面级子任务。
- 每个子任务都有一个路由：`modify_existing_page`、`add_module` 或 `add_page`。
- 修改已有页面会跳过方案图生成。
- 新增模块和新增页面必须生成方案图并等待确认。
- 目标不明确时，已先让用户确认。

## 方案图检查

用于新增模块和新增页面。

- 方案图由 image gen 指令直接生成。
- 方案图源文件是 PNG、JPEG 或 WebP 等位图。
- 方案图不是 SVG、HTML/CSS、Canvas、Figma shape 脚本、手写矢量或其他代码绘图产物。
- 没有把 SVG 或代码图导出/截图成 PNG 后当作 image gen 方案图。
- 运行记录已写明 `proposalImage.outputType`，且结果为位图。
- 方案图符合用户需求。
- 方案图体现了可用的产品级或页面级设计规范。
- 内容层级清晰。
- 没有引入无关 UI 文案。
- 视觉方向能够被还原为可编辑 Figma 结构。
- 已明确该方案图是后续 Figma 写入的视觉还原标准，而不是仅供“灵感参考”。
- 已从方案图提取视觉锚点清单：模块高度/比例、主 CTA 尺度、装饰元素、图标/插画质感、信息层级、留白节奏和视觉重心。
- 用户已明确确认方案图后，才进入 Figma 写入。

## 装饰素材检查

仅在装饰图像需要作为独立 Figma 素材时使用。

- 已对照本次 image gen 方案图逐项识别装饰视觉，而不是只根据文字需求判断。
- 轻拟物 icon、质感券卡、皇冠、会员徽章、插画物件、复杂光效、产品 mockup、纹理和高识别度视觉元素均已按装饰素材处理。
- image gen 方案图中的复杂装饰、水印、光效、质感图形、特殊插画或难以用基础图层高保真复现的元素，必须导出或重新生成为独立透明 PNG；不能用随手画的 polygon、rectangle 或简化 SVG 替代。
- 只要装饰元素来自已确认方案图，就不允许在 Figma 还原阶段用可编辑基础图层重新绘制来替代；必须使用确认方案图裁图/重生成得到的透明 PNG。
- 如果装饰元素来自方案图，必须先裁出参考元素，再基于该裁图用 image gen 重新生成高清绿底源图；禁止直接把低清参考图裁片作为最终素材。
- 绿底高清源图必须使用 chroma-key 抠图得到透明 PNG，并保留源图、透明 PNG 和校验图路径。
- 抠图后必须在红、绿、黑等极端背景下合成校验，确认主体完整、边缘清晰、无绿边、无暗边、无背景残留。
- 该素材确实属于装饰元素，而不是更适合用可编辑 Figma 图层表达的内容。
- 素材背景透明。
- 浅色背景下边缘干净。
- 深色背景下边缘干净。
- 极端色背景下边缘干净。
- 没有残留无关 UI 碎片。
- 光效、阴影和透明度看起来合理。
- 最终格式为带 alpha 的 PNG。

## Figma 还原检查

在 `use_figma` 写入后运行。

### 页面级规范硬性机械校验

适用于已有页面新增模块和局部布局匹配。该校验优先级高于方案图观感、个人判断和口头说明；未通过时 `designSystem` 和 `specConstraintsApplied` 必须记为 `fail`，返回 `read_context` 或 `write_figma` 修复后重跑。

校验原则：

- 必须先读取 [figma design agent 页面级设计规范.md](figma%20design%20agent%20%E9%A1%B5%E9%9D%A2%E7%BA%A7%E8%AE%BE%E8%A7%84%E8%8C%83.md)，定位目标页面章节，例如 `Page02·列表页`、`Page03·详情页` 或 `Page04·下单页`。
- 必须把目标页面章节中的硬性数值提取到运行记录 `pageSpecConstraints`，不能只记录“已读取规范”。
- 校验必须以 Figma 节点属性、变量绑定、截图量测或运行脚本输出为证据；机械验证优先，目测只能作为补充。
- 页面级规范与方案图冲突时，页面级规范胜出；不允许为了贴近方案图而改动规范中明确给出的字号、颜色、间距、内外边距、圆角、描边或阴影。
- 未能读取或量测关键属性时，该项按未通过处理，不能用“看起来接近”替代。

必须机械核验并记录证据：

| 类别 | 校验项 | 通过条件 |
|------|--------|----------|
| 页面命中 | 目标页面章节 | 运行记录写明使用的页面章节和对应 Figma 原页面/节点 |
| 尺寸 | 模块宽度、高度规则 | 宽度符合页面规范；高度为内容自适应且无裁切 |
| 圆角 | 模块整体、内卡片、标签 | 与页面规范数值完全一致，例如 24px、12px、4px |
| 底色/渐变 | 模块背景 | 使用页面规范指定纯色或渐变；固定白底页面不得使用其他底色 |
| 内边距 | 上、右、下、左 | 与页面规范一致；“均24px”必须四边均为 24px |
| 外边距 | 左右、上下 | 页面规范有外边距时必须逐边核验，例如 0px、18px |
| 元素间距 | 独立内容、基础间距、同类重复 | 只使用页面规范允许的 24px、18px、12px、6px 等间距 |
| 字号 | 标题、正文、辅助文字 | 分别符合 30px、28px、24px，或页面章节指定值 |
| 文字颜色 | 标题、正文、辅助文字 | 分别符合 `#0F131A`、`#5C5F66`、`#919499`，或绑定到等价规范变量 |
| 字体/字重 | 文本样式 | 使用页面规范或 token 指定文本样式，不得随意降级 |
| 描边 | 模块级卡片 | 页面规范写明不使用描边时，stroke 必须为空或 0 |
| 阴影 | 模块级卡片 | 页面规范写明不使用阴影时，effects 必须为空 |
| 固定布局 | 页面特殊规则 | 例如下单页标题距模块左/上均 24px，必须单独量测 |

证据要求：

- 运行记录必须包含 `specConstraintsApplied` 的逐项结果，而不是单个总括结论。
- 每个失败项必须写明：规范值、Figma 实际值、目标节点、修复动作。
- 对颜色必须记录实际 hex 或变量名；对字号、圆角、padding、gap、margin 必须记录 px 数值。
- 使用截图时，应只作为“布局、裁切、重叠、视觉差距”的辅助证据；字号、颜色、间距、内外边距、圆角优先从节点属性或脚本量测。

内容：

- 所有必需文案和内容都已出现。
- 没有占位文本残留。
- 画布里没有需求说明。

设计规范：

- 对新增模块，必须按“页面级规范硬性机械校验”逐项核对页面级规范的具体数值：模块尺寸、模块圆角、模块内卡片圆角、标签圆角、模块底色/渐变、内外边距、模块内元素间距、标题/正文/辅助字号、标题/文本/辅助颜色、无模块描边、无阴影和页面特殊固定布局。
- 如果 image gen 方案图与页面级规范冲突，必须以页面级规范为硬约束，只保留方案图的信息层级和视觉意图。
- 不允许用“更接近方案图”为理由改大/改小页面级规范中已经明确的字号、圆角、间距或底色。
- 颜色使用已批准变量，或使用最接近的规范内颜色。
- 字体、字号、行高和字重符合产品规则。
- 合适位置复用了组件。
- 圆角、间距、栅格和密度符合产品/页面规则。

布局：

- 内容驱动区域使用自动布局。
- frame 没有裁切内容。
- 文本没有重叠或溢出。
- 模块高度适应内容。
- 响应式约束符合目标屏幕。

视觉：

- 修改已有页面时，周边结构保持稳定。
- 新增模块/页面与已确认方案图方向一致。
- Figma 写入前已打开并对照确认方案图；禁止只按文字需求、规范数值或个人判断直接写入。
- Figma 写入过程中已按方案图视觉锚点逐项还原，不能只保证文案齐全或大致同色。
- 写入 Figma 后必须把 Figma 截图与本次 image gen 方案图并排比对，检查模块高度、主 CTA 尺度、装饰元素、信息层级、留白节奏和视觉重心；不能只检查“内容是否齐全”。
- 并排对比图已保存到运行记录；交付时需要说明参考图与设计稿差距是否过大。
- 如果核心视觉锚点缺失或明显弱化，必须先修复再标记视觉通过。
- 如果 Figma 设计稿与确认方案图在模块比例、卡片布局、CTA 样式、装饰元素、视觉重心或整体质感上差距过大，视觉检查必须判定为 fail，并返回 `write_figma` 修复。
- 最终结果看起来属于当前产品，而不是外来设计。

素材：

- 装饰 PNG 素材边缘干净。
- 已单独比对每个装饰 PNG 与确认方案图中的对应视觉元素，检查形状、构图、光感、质感、比例和位置一致性。
- Figma 中的装饰元素来自通过校验的透明 PNG，不是基础图层重画、手绘 SVG 或未校验裁片。
- 位图只在合适场景使用。
- 没有把整页截图作为最终设计。

## 场景测试

### 修改已有按钮文案

预期：

- 路由：`modify_existing_page`。
- 读取目标模块上下文。
- 跳过 image gen。
- 只更新 Figma 中的按钮文案。
- 自检确认没有布局回归。

### 新增首页模块

预期：

- 路由：`add_module`。
- 读取所在页面上下文和页面级规范。
- 生成方案图。
- 等待用户确认。
- 还原为可编辑 Figma 模块。
- 自检确认布局与设计规范合格。

### 新增完整页面

预期：

- 路由：`add_page`。
- 读取产品级规范。
- 生成方案图。
- 等待用户确认。
- 还原为可编辑 Figma 页面。
- 自检确认页面层级和产品一致性。

### 方案图包含装饰光效

预期：

- 识别光效为装饰素材。
- 导出或重新生成透明 PNG。
- 在浅色和深色背景上检查。
- 放入 Figma 时不转换为 SVG。
- 交付前将光效/装饰 PNG 与方案图原始视觉单独比对。

### 方案图包含轻拟物图标或质感物件

预期：

- 识别轻拟物图标或质感物件为装饰素材。
- 从确认方案图裁出对应元素，按素材流程生成或处理为透明 PNG。
- 上传透明 PNG 到 Figma，并放入对应卡片或模块位置。
- 禁止在 Figma 中用基础 shape/vector 重新绘制替代。
- 交付前保存确认方案图与 Figma 设计稿并排对比图，并检查装饰元素一致性。

### 方案图使用非规范色

预期：

- 不盲目复制方案图原始颜色。
- 选择最接近的产品规范色。
- 在符合规范的前提下保留视觉意图。

### 自检发现画板裁切

预期：

- 修复 frame 尺寸、约束或自动布局。
- 重新运行自检。
- 问题解决后再交付。
