# 推演能力路由

本文件用于在正式进入业务状态推演前，判断当前需求主要调用哪一种补全能力，并选择对应规则检查清单。目标是按“推演能力”组织推理，而不是按页面表面相似度套用旧样本。

## 使用时机

每次使用 `design-state-completion` 时，在阶段 1 前先执行能力路由。

输出必须包含：

- 主能力：只能选择 1 个。
- 辅助能力：最多选择 1 个，可为空。
- 判断依据：说明为什么命中该能力。
- 规则检查清单：列出本次必须检查的对象、组件、字段或布局风险。
- 不适用能力：说明哪些能力不适用，避免推演发散。

内部调试资产和调试期标准答案边界以 `execution-pipeline.md` 为准；本文件只负责能力命中和检查重点。

## 能力分类

### 1. 状态矩阵推演

适用场景：

- 用户要求补齐业务全情况、全状态、状态矩阵。
- 需求重点是识别业务变量、合法组合和不适用组合。
- 页面或模块状态由多个业务变量组合决定。

检查重点：

- 核心业务对象是否明确。
- 矩阵主轴是否来自业务变量，而不是 UI 文案。
- 是否过滤非法组合和低价值组合。
- 已完成主态是否进入矩阵。

常见误用：

- 把平台基础异常态当作主要矩阵。
- 只列状态名，没有触发条件和用户含义。

### 2. 子组件变体补全

适用场景：

- 用户要求“生成为子组件”或补 component / component set。
- 状态差异主要发生在卡片、权益项、挑战卡、rate、标题区、按钮区等子结构内。
- 复合模块需要先补子组件系统，再组合页面状态。

检查重点：

- 原组件、实例、component set 和 variant properties 是否已识别。
- 补充变体是否进入同一个父 component set。
- 哪些状态由 variant 切换覆盖，哪些需要结构扩展。
- 是否避免手绘近似稿替代原组件。

常见误用：

- 把整页场景误压缩成组件变体。
- 把普通 hover / pressed / focused 当作业务状态。

### 3. 页面组合态生成

适用场景：

- 状态必须放回完整页面才能验证信息层级、CTA、上下游链路或模块承接。
- 子组件或模块状态已确定，但需要看它在真实页面中的组合效果。
- 页面级场景会改变导航、tab、列表数量、模块位置或主行动。
- 用户提供的是完整页面节点，且目标模块高度、数量或显隐变化会影响下方页面流内容、浮层标签、列表项位置或页面可见区域。

检查重点：

- 是否复制真实页面或真实模块实例。
- 页面级状态是否用完整页面呈现。
- CTA、信息层级和上下游链路是否成立。
- 模块状态放回页面后是否仍符合业务语义。
- 是否建立后续页面流对象清单，包括列表项、依附标签、浮层、吸底栏和其他跟随模块位置变化的对象。
- 模块高度变化时，是否明确下方内容是保持原位还是按页面流顺延；若用户要求与合理页面流冲突，是否先确认。

常见误用：

- 对纯子组件矩阵生成过多完整页面。
- 用缩略线框替代完整页面状态。

### 4. 内容边界布局

适用场景：

- 核心变化来自数量、文案长度、图片缺失、展示上限或信息密度。
- 用户关注 0 / 1 / 多个 / 超出上限等内容边界。
- 页面结构不一定变，但布局密度和承载方式会变。

检查重点：

- 数量边界是否完整：0、1、2、多个、超过展示上限。
- 长文案、缺图、缺标签、字段缺失是否覆盖。
- 布局是否智能适配，而不是机械拉伸、删除或缩放。
- 固定底 bar、悬浮入口、安全区是否被误移动。

常见误用：

- 把内容数量变化只当作文案差异。
- 默认保留前 N 个对象，不判断业务优先级。

### 5. 跨模块表达迁移

适用场景：

- 已有一个模块或对象类型完成表达，需要迁移到另一个模块或对象类型。
- 例如房型 rate 的权益表达迁移到普通 rate。
- 重点是抽取表达规则，而不是复制来源结构。

检查重点：

- 源模块表达规则是什么。
- 目标模块的等价承载字段在哪里。
- 哪些表达可以迁移，哪些因结构或业务语义不同不能迁移。
- 迁移后是否保持目标模块原有密度和组件语义。

常见误用：

- 直接复制来源模块结构破坏目标模块。
- 只迁移视觉样式，没有迁移业务触发条件。

### 6. 平台基础状态处理

适用场景：

- 用户明确点名 loading、骨架屏、网络失败、空态、权限失败等平台基础状态。
- 任务目标是补齐平台基础规范中的状态，而不是业务全情况。

检查重点：

- 用户是否明确要求处理平台基础状态。
- 未明确点名时，是否只在矩阵中标注“由平台基础规范覆盖”。
- 基础状态是否套用平台规范，不扩展成业务状态。

常见误用：

- 用户没有要求时主动生成 loading / error / empty 画稿。
- 把平台异常态和业务状态矩阵混在一起。

```md
- 不适用能力：
```

## 失败归因规则

复盘时先判断失败属于哪一种能力，再决定沉淀位置。

- 状态矩阵推演失败：变量遗漏、组合未过滤、矩阵主轴错位。
- 子组件变体补全失败：组件结构拆错、variant 归属错误、手绘替代实例。
- 页面组合态生成失败：模块状态放回页面后 CTA、层级或链路不成立。
- 内容边界布局失败：数量、长文案、缺图、展示上限或密度处理错误。
- 跨模块表达迁移失败：复制来源结构、目标字段落点错误、不可迁移项未过滤。
- 平台基础状态处理失败：未点名却生成基础状态，或点名后没有按平台规范处理。
