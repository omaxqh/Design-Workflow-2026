# Phase 8 — 测试 / 评估

> 关联章节：architecture-review §7.7

---

## T8.1 Golden Test Suite

**状态**: 未开始 | **优先级**: P1 | **关联痛点**: H27

改 skill 后没东西跑回归。需要为每个 skill 建立 golden test case（固定输入 → 期望输出范围），每次 skill 变更必须跑通。

**交付物**: golden test 框架 + 每个 skill 至少 3 个 golden case + CI 集成

---

## T8.2 回归检测

**状态**: 未开始 | **优先级**: P1 | **关联痛点**: H28

skill 变更不该恶化已有 case。对比变更前后的输出质量分，显著下降 → 阻止合并。

**交付物**: 质量分对比脚本 + 阈值配置 + PR check 集成

---

## T8.3 A/B 评估框架

**状态**: 未开始 | **优先级**: P2 | **关联痛点**: H29

怎么知道 skill 改动确实更好？需要对同一组输入跑两个版本，人工盲评或自动评分。

**交付物**: A/B 评估脚本 + 评分维度定义 + 结果报告模板
