## T-RESEARCH.1 硬解 vs 软解 设计生成方案调研

**状态**: 未开始  
**优先级**: P1 — 不阻塞主线，可与开发并行  
**产出影响**: T3.3（S3 设计）

### 目标

用户明确要求：真实测试后做调研，硬解和软解各自是否有更好的方案。

### 调研方向

**硬解方向**（IDE + 设计 skill + 设计规范 → 直接生成 Figma）：
- pencil / googlestitch 类工具现状
- 大厂场景下是否有"稳定且不丑"的方案
- 约束：必须遵守设计规范 token，不能随意发挥

**软解方向**（image2 出图 → 转 Figma 图层 → 人微调）：
- 当前最佳 image2 模型对比（GPT Image 2 / Midjourney / 其他）
- 图 → Figma 可编辑图层的转换方案
- 保真度与可编辑性的 trade-off

### 评估标准

| 维度 | 权重 |
|------|------|
| 输出稳定性（同输入多次跑，结果一致性） | 高 |
| 设计规范遵守度 | 高 |
| 美感（vs 设计师手作） | 中 |
| 可编辑性（设计师能方便微调） | 高 |
| 成本（token / API 调用） | 低 |
