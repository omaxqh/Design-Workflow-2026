## 一、体验评估四维度

### 1. 链路连贯性
`标签: 流程、路径、步骤、跳转、一致性`

| 规则 | 量化标准 | 判断方法 |
|------|----------|----------|
| 核心流程零断裂 | 关键任务完成路径 ≤ 3 步 | 数步骤数，跳转即算一步 |
| 操作路径一致 | 同类操作在不同页面位置偏差 ≤ 1 层级 | 对比同类元素在多个页面的位置 |
| 上下文不丢失 | 跨页面时关键信息保留率 = 100% | 检查跳转后是否能看到之前选择 |

**每多一步操作，流失率提升 10-20%**

✅ Good：列表页点击 → 详情页 → 下单页（3步完成预订）  
❌ Bad：列表页 → 详情页 → 选日期页 → 选房型页 → 填信息页 → 确认页（6步）

---

### 2. 信息表达完整性
`标签: 信息缺失、决策信息、价格透明、规则说明`

| 规则 | 量化标准 | 判断方法 |
|------|----------|----------|
| 决策信息可见 | 用户做决策时，必要信息可见率 = 100% | 列举决策所需信息，逐一检查 |
| 价格透明 | 最终价格与初始展示偏差 ≤ 5% | 对比首次看到的价格 vs 支付价 |
| 规则前置 | 限制条件在操作前展示，非操作后弹窗 | 检查限制条件出现时机 |

**用户做决策的那一刻，所有必要信息必须在视野内**

✅ Good：房型卡片上直接显示「不可取消 | 含早 | 大床」  
❌ Bad：下单后弹窗告知「该房型不可取消」

---

### 3. 文案可读性
`标签: 文案、标题、按钮、利益点、可读性`

| 规则 | 量化标准 | 判断方法 |
|------|----------|----------|
| 标题表达利益点 | 标题含具体利益词（数字/动作/结果） | 去掉标题后用户是否仍能理解价值 |
| 文案长度控制 | 标签 ≤ 8字，说明 ≤ 20字，标题 ≤ 15字 | 字数统计 |
| 零歧义 | 同一文案不可有两种理解 | 找3个人读，理解是否一致 |
| 避免术语 | 不使用业务内部术语 | 检查是否有非用户语言 |

**5秒规则：用户5秒内看不懂就会放弃**

✅ Good：「V10 多得免费早餐」—— 具体、有利益、8字  
❌ Bad：「尊享V10专属权益服务」—— 抽象、无利益点、不知道是什么

---

### 4. 交互好用性
`标签: 层次、操作、按钮、复杂度、认知负担`

| 规则 | 量化标准 | 判断方法 |
|------|----------|----------|
| 主操作唯一突出 | 一屏内主按钮 = 1个 | 视觉扫描，是否一眼定位 |
| 信息分层 | 一屏信息块 ≤ 5个 | 数独立信息区块数 |
| 操作可预期 | 点击后结果与预期一致率 = 100% | 按钮文案是否暗示了结果 |

✅ Good：页面底部一个「立即预订」按钮，颜色醒目  
❌ Bad：页面有「预订」「咨询」「收藏」「分享」4个同等大小按钮

## 二、设计决策优先级框架

`标签: 优先级、排序、取舍`

| 优先级 | 维度 | 判断标准 | 举例 |
|--------|------|----------|------|
| P0 | 功能可用性 | 用户无法完成核心任务 | 支付按钮不可点击 |
| P1 | 信息完整性 | 关键信息缺失→误解/客诉 | 不可取消规则未展示 |
| P2 | 效率优化 | 能完成但路径冗长 | 需要5步才能到支付 |
| P3 | 体验打磨 | 视觉/文案/动效细节 | 按钮圆角不统一 |

**决策规则**：当两个方案冲突时，优先级高的维度胜出。同优先级时选实现成本低的。

## 三、设计启发式（场景化）

### 认知负荷管理
`标签: 信息量、选项、分步、折叠`

| 场景 | 规则 | 阈值 |
|------|------|------|
| 列表页每个卡片 | 关键信息点 | ≤ 4个（名称+价格+1标签+1评分） |
| 详情页一屏 | 独立信息区块 | ≤ 5个 |
| 选项选择 | 同层级可见选项 | ≤ 5个，超出折叠 |
| 多步流程 | 每步操作数 | = 1个核心决策 |
| 权益/标签展示 | 可见权益标签 | ≤ 3个，超出「+N项」 |

---

### 视觉层次
`标签: 层级、强调、留白、对比`

信息权重从高到低：
1. **核心行动**（按钮/价格）— 最大字号 + 品牌色
2. **关键数据**（评分/折扣）— 加粗 + 强调色
3. **标题/名称** — 标准字号 + 深色
4. **说明文字** — 次号 + 灰色
5. **辅助信息** — 最小号 + 浅灰

**间距规则**：不同区块间距 > 区块内元素间距 × 2

---

### 用户心理原则
`标签: 心理、动机、说服、文案策略`

| 原则 | 适用场景 | 执行方式 | 禁忌 |
|------|----------|----------|------|
| **差异感知** | 权益/会员/升级 | 用对比框架：展示「你比别人多了什么」 | ❌ 不用压力式（"不用就亏了"） |
| **锚定效应** | 价格/套餐选择 | 先展示原价/高价方案，再展示实际价 | ❌ 不虚构原价 |
| **社会认同** | 决策犹豫场景 | 展示「XX人已预订」「好评率98%」 | ❌ 不伪造数据 |
| **确定性偏好** | 不确定结果场景 | 明确告知状态+预计时间 | ❌ 不用模糊表述（"可能""也许"） |
| **庆祝感** | 获得权益/完成行动 | 正向反馈：「已为你节省¥XX」 | ❌ 不用损失框架（"错过就没了"） |

> 💡 项目级的文案策略决策（如「差异感知+庆祝感」替代「损失厌恶+紧迫感」）请查看 `BussinessExp.md` 中的项目归档。

## 四、反模式清单

> 💡 行业经验（旅行/会员/电商）已迁移至 `BussinessExp.md`，请在 Step 3 审核 PRD 时同步读取。

`标签: 避免、禁止、反面教材、设计审核`

| 反模式 | 识别信号 | 替代方案 | 严重度 |
|--------|----------|----------|--------|
| 信息过载 | 一屏 > 7个信息块 | 分层折叠，默认展示 ≤ 5个 | P2 |
| 虚假紧迫 | 倒计时无真实数据支撑 | 用真实库存/真实截止时间 | P1 |
| 暗黑模式 | 取消/退出按钮被弱化 | 所有操作平等可见 | P0 |
| 决策瘫痪 | 同级选项 > 5个 | 默认推荐 + 折叠其余 | P2 |
| 认知断裂 | 跳转后丢失之前选择 | 关键上下文始终可见 | P1 |
| 压力文案 | 「不用就过期」「错过就没了」 | 庆祝感文案：「已为你准备」 | P2 |
| 模糊状态 | 「处理中」无时间预期 | 明确状态+预计完成时间 | P1 |

## 五、评分标准

`标签: 评分、审阅、评估`

每个维度评分 0-10：

| 分数 | 水平 | 判断依据 |
|------|------|----------|
| 9-10 | 标杆 | 满足所有规则 + 有创新亮点 |
| 7-8 | 良好 | 满足所有规则，无创新 |
| 5-6 | 及格 | 有 1-2 条规则未满足，但不影响核心任务 |
| 3-4 | 较差 | 有规则未满足，影响用户体验 |
| 0-2 | 不可接受 | 核心任务受阻或有反模式 |

**扣分规则**：
- 存在 P0 反模式 → 该维度直接 ≤ 2
- 存在 P1 反模式 → 该维度 ≤ 5
- 文案量化不达标 → 文案可读性维度 -2
- 步骤超标 → 链路连贯性 -2/超出步
