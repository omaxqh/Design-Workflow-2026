# 88VIP 表达升级 — 设计需求文档（DRD）

> **来源 PRD**：https://aliyuque.antfin.com/awu.wj/osxlhg/az3lkmvnzpl99vwa
> **产出时间**：2026-05-11
> **Pipeline**：SmartDesigner 01_PRD to DRD · Step 1 → Step 2 → Step 4 → Step 5
> **设计师拍板记录**：5 个关键设计点均已通过 IDE 选项面板一次性拍板

## 一、核心策略

**一句话定位**：把 88VIP 付费会员身份从「分散稀释在 F 等 / 二官 / 锦江三套会员体系」升级为「全链路一致可见、核心优惠合并锚定、权益聚焦教育」。

**设计关键词**：身份可见、合并锚定、主次分层、跨页一致、规则前置

---

## 二、关键决策点

### 决策 1 · 列表营销标合并表达 —— icon 自带金额锚点

- **推荐**：A · icon 直接展示预估省额 + 点击看明细
- **原因**：88VIP 老用户的第一认知诉求是"我这身份到底省了多少钱"，合并 icon 若只做身份标签、金额藏浮层，会让付费身份价值被稀释（`BE 一.会员权益·对比框架` + `DP 一.2 决策信息可见`）
- **具体方案**：
  - 异形 icon 标签文字为「88VIP 专享优惠 · 约省 ¥XX」
  - 点击 icon 弹浮层拆解「88VIP 专享价 ¥A + 旅行消费券 ¥B = 合计省 ¥XX」
  - 金额字段由营销链路预估返回，文案必须带「约」字，规避客诉

### 决策 2 · 全链路免早表达 —— 身份前缀统一模板

- **推荐**：A · 「[身份] · X 份早餐」模板 + 房型浮层升级用直观份数对比
- **原因**：88VIP 表达升级的本质是"让身份可感知"；现行 4 套文案（88VIP 可兑 / Fx 可兑 / 金卡会员可享 / 金卡享 x 份）跨页跨身份认知断裂，命中 `DP 四·认知断裂` 反模式
- **具体方案**：
  - 全链路（列表 / 详情 / 下单 / 订详 / 房型浮层）统一「[身份] · X 份早餐」
  - 88VIP 用户看到「88VIP · 1 份免费早餐」；锦江金卡看到「金卡 · 1 份早餐」；Fx 用户看到「Fx · 1 份免费早餐」
  - 锦江房型浮层升级样式用「升级前 1 份 → 升级后 2 份」直观对比，不再用 rate-1 / rate 数学差值
  - 前端须把"身份+份数"变量封装为通用组件，禁止各端自由拼接

### 决策 3 · 首页 88VIP 权益组件 —— 主次分层

- **推荐**：A · 首屏大卡推 2 个当期核心权益 + 其余 5 个折叠到「+5 项更多权益 →」
- **原因**：7 个权益点并列命中 `DP 三·认知负荷`（权益标签 ≤ 3 个）和 `BE 一·电商首屏 3 秒`，用户看了也记不住任何一个；主次分层既符合认知阈值又保留全部权益的触达路径
- **具体方案**：
  - 首屏大卡展示 2 个运营配置的当期核心权益（主标题 + 副标题 + 大 icon + 跳转链）
  - 下方 "+5 项更多权益 →" 按钮，点击弹 88VIP 全权益浮层（展示全部 7 项）
  - 运营后台支持日/周维度调整主推权益；兜底默认："88VIP 专享价 + 闪购早餐券"

### 决策 4 · 注册红包「不算价」—— 保留 PRD 原文案 + 浮层承接规则

- **推荐**：B · 标签保持「注册下单再减 ¥XX」；浮层说明抵扣规则
- **原因**：设计师优先保留 PRD 的营销表达力，规则说明由浮层承接
- **具体方案**：
  - 列表营销标文案维持 PRD 原设计「注册下单再减 ¥XX」
  - 点击/展开时在优惠明细浮层中必须包含一条明确说明：「此金额需在下单页完成注册后抵扣 · 当前列表价不含」
  - 浮层说明文案必须在第一眼可见（不可折叠在二级入口）
- **⚠️ 残留风险**：此方案命中 `DP 四·虚假锚点`（P1 反模式），浮层兜底不彻底时仍有用户投诉风险 —— 登记在四·风险提示

### 决策 5 · 会员卡片已开通态 —— 88VIP 异形角标

- **推荐**：B · 右上角挂 88VIP 异形角标；主文案维持 PRD 新逻辑
- **原因**：PRD 新逻辑删除了身份前缀，会让 88VIP 用户在会员卡片看不到付费身份，与全链路身份强化矛盾；角标是信息密度（主文案不变）和身份可见（角标承载）的最佳平衡（`DP 三·视觉层次` + `BE 一·会员权益`）
- **具体方案**：
  - 88VIP 用户的已开通会员卡片右上角挂 88VIP 异形角标（异形·非方形，承接 88VIP 品牌色）
  - 主文案维持 PRD 新逻辑「[商家名称] · [商家会员等级]」不变（非 88VIP 同步遵循）
  - 未开通态 88VIP 用户继续用 PRD 新逻辑「[用户F等] × [商家名称] [建议开通等级]」
  - 首次浏览增加轻动效引导，提升角标识别度

## 三、文案规范

| 场景 | 推荐文案 | 避免文案 | 原则依据 |
|------|----------|----------|----------|
| 列表合并营销 icon | 88VIP 专享优惠 · 约省 ¥XX | 88VIP 专享优惠（无金额锚点）| `DP 一.2 决策信息可见` |
| 列表合并 icon 浮层 | 88VIP 专享价 ¥A + 旅行消费券 ¥B = 合计省 ¥XX | 只露 icon 不给拆解 | `DP 一.2 信息完整性` |
| 免早 · 88VIP | 88VIP · 1 份免费早餐 | 88VIP 可兑免费早餐 | `DP 一.3 零歧义` |
| 免早 · 锦江金卡 | 金卡 · 1 份早餐 | 金卡会员可享免费早餐 / 金卡享 1 份早餐 | `DP 一.3 零歧义` |
| 免早 · 锦江白金卡 | 白金卡 · 2 份早餐 | 白金卡享 x 份早餐 | `DP 一.3 零歧义` |
| 免早 · Fx | Fx · 1 份免费早餐 | Fx 可兑免费早餐 | `DP 一.3 零歧义` |
| 免早 · 二官金卡 | 金卡 · 本人入住 1 份早餐 | xx 会员本人入住可享免费早餐 | `DP 一.3 避免啰嗦 + 零歧义` |
| 房型浮层升级样式 | 升级前 1 份 → 升级后 2 份 | rate-1 份 / rate 份（数学差值）| `DP 一.2 规则前置` |
| 首页权益组件 | 2 个大卡 + "+5 项更多权益 →" | 7 个并列小 icon | `DP 三·认知负荷` |
| 不算价红包标 | 注册下单再减 ¥XX（+ 浮层第一屏说明抵扣规则）| 注册下单再减 ¥XX（无浮层兜底）| `DP 一.2 规则前置` |
| 详情页营销标 88VIP | 88VIP 免费早餐 | Fx 免费早餐（对 88VIP 用户）| `BE 一·会员权益身份对比` |
| 下单页本单可享标题 | 88VIP 本单可享 / xx 本单可享 | 仅"本单可享"不带身份 | `BE 一·会员权益·身份对比` |
| 订详页本单可享角标 | 88VIP | Fx（对 88VIP 用户） | `BE 一·会员权益` |
| 会员卡片未开通·非锦江 | [用户F等] × [商家名称] [建议开通等级] | [用户F等] 直通 [商家名称] [等级] 权益（PRD 原逻辑「直通」）| `DP 一.3 避免术语` |
| 会员卡片未开通·锦江 | [用户淘宝身份] × [商家名称] [建议开通等级] | [用户淘宝身份] 直通 [...] 权益 | `DP 一.3 避免术语` |
| 会员卡片已开通·88VIP | [商家名称] · [商家会员等级]（+ 右上角 88VIP 异形角标）| 88VIP · 商家名称 · 商家会员等级（三段主文案）| `DP 三·视觉层次` |
| 会员卡片已开通·非 88VIP | [商家名称] · [商家会员等级] | — | `DP 一.3 简洁` |

**文案全局原则**：

1. **身份前缀统一**：所有涉及身份表达的场景使用「[身份] · 内容」模板，`·` 是固定分隔符，禁止用「享/可兑/可享」等动词
2. **金额锚点前置**：合并营销表达必须在 icon 上就让用户看到金额，浮层只做拆解不承担"初次告知"职责
3. **不算价文案必须带浮层说明**：营销标文案可保持简短，但浮层首屏必须明确抵扣时点，不能藏在二级入口
4. **房型升级用直观对比**：禁止用数学差值表达份数变化（rate-1 / rate 数值已在设计层拒收），统一用「升级前 X → 升级后 Y」

## 四、风险提示

| 风险 | 场景 | 规避方案 |
|------|------|----------|
| **虚假锚点（P1 反模式）** | 注册红包"不算价"文案用「注册下单再减 ¥XX」让用户误以为是到手价（决策 4 保留 PRD 方案带入的残留风险）| 1. 浮层说明必须在首屏可见，不可折叠<br>2. 文案末尾建议补 `（限注册抵扣）` 微缩说明<br>3. 上线后跟踪"不算价导致的退改率"，高于基线 5% 触发文案迭代 |
| **预估金额偏差客诉** | 列表合并 icon「约省 ¥XX」基于营销链路预估，与下单页实际金额可能有偏差 | 1. 文案必须带「约」字<br>2. 浮层必须展示真实拆解 + 去下单页查看实际金额的引导<br>3. 业务 + 法务须确认预估算法的偏差上限（建议 ≤ 10%）|
| **运营核心权益高频决策** | 首页 88VIP 组件只推 2 个核心，运营需判断哪 2 个当期最强 | 1. 提供运营后台配置（日/周维度调整）<br>2. 默认兜底："88VIP 专享价 + 闪购早餐券"<br>3. 提供 A/B 能力验证主推权益点击率 |
| **角标识别度弱** | 会员卡片右上角 88VIP 角标可能被忽略（决策 5）| 1. 角标必须用异形（非方形）+ 88VIP 品牌色<br>2. 首次访问加 1 次轻动效引导（0.8-1.2s 缩放）<br>3. 跟踪角标点击率 / 曝光率 |
| **跨端模板不一致** | 身份前缀统一模板需五端 + 房型浮层全改，任一端漏改导致认知断裂 | 1. 前端封装通用身份组件 `<IdentityLabel identity="88VIP" content="1 份免费早餐" />`<br>2. 禁止各业务端自由拼接身份字符串<br>3. 组件层做测试覆盖 |
| **合并 icon 偏差挑战业务合规** | 「约省 ¥XX」如果用户下单后实际少于预估，涉及营销广告法风险 | 法务锁定文案样式；「约」字 + 浮层免责声明不可删 |
| **首页权益组件"+5 项"折叠的曝光下降** | 未入选的权益点首屏曝光显著降低 | 浮层内支持"看全部权益"按钮；权益曝光率由运营后台监控 |

## 五、优先级排序

| 优先级 | 设计任务 | 理由 |
|--------|----------|------|
| **P0** | 列表营销标合并 icon 带金额锚点（决策 1）| 88VIP 核心感知入口，直接影响付费身份价值传达，列表是用户第一触达场 |
| **P0** | 全链路免早表达身份前缀模板（决策 2）| 跨五端认知一致性是本期升级根基；任何一端漏改认知就断 |
| **P0** | 首页 88VIP 权益组件主次分层（决策 3）| 首页是 88VIP 身份教育的第一触点，7 并列会失效 → 整个首页教育链路失效 |
| **P1** | 会员卡片已开通态 88VIP 角标（决策 5）| 身份可见性细节补强，不做不影响主流程但影响身份一致性 |
| **P1** | 不算价红包浮层说明（决策 4）| 规避 P1 反模式（虚假锚点），防止客诉升级 |
| **P1** | 下单页 / 订详页本单可享 88VIP 优先（PRD F6/F7）| 下单和订详是决策-履约闭环的关键页，身份统一表达 |
| **P2** | 详情页早餐政策样式颜色加强（PRD 原要求）| 视觉层（L2 Token）加强，排在内容决策之后 |
| **P2** | 综合筛选浮层新增 88VIP 专享价筛选项（PRD F3）| 功能点，非感知升级主线 |
| **P2** | 二官黑标 icon 更改（PRD F8）| 图标层，跟随会员卡片改造一起上线 |

## 六、设计规格

### 触点 1 · 首页 · 88VIP 酒店专享权益组件

**触发条件**：用户为 88VIP 身份（人群 ID 126835 命中）+ 境内酒店飞猪首页

**新增/改动元素**：

| 元素 | 位置 | 结构 | 内容规则 | 约束 |
|------|------|------|----------|------|
| 组件标题 | 组件顶部 | 文字 + 88VIP 品牌色小 icon | 写死「88VIP 酒店专享权益」| ≤ 12 字 |
| 主推权益大卡 | 标题下方 | 2 个并列大卡（icon + 主标题 + 副标题） | 运营配置的 2 个当期核心权益 | 每卡主标 ≤ 10 字，副标 ≤ 16 字 |
| 更多权益按钮 | 大卡下方 | 胶囊按钮 + `→` 箭头 | 「+5 项更多权益 →」| 固定文案 |
| 全权益浮层 | 底部半屏浮层 | 7 个权益点列表（icon + 主标题 + 副标题 + 跳转）| 运营配置的全部 7 项权益 | 每项 ≤ 30 字描述 |

**状态枚举**：

| 状态 | 触发条件 | 展示内容 | 交互行为 |
|------|----------|----------|----------|
| 默认态 | 88VIP 用户进入首页 | 组件常态展示 2 个主推大卡 + 按钮 | 大卡点击跳对应权益详情；按钮点击弹浮层 |
| 浮层展开 | 用户点击 "+5 项更多权益" | 半屏浮层展示全 7 项 | 点击任一项跳对应权益详情；点击关闭回首页 |
| 非 88VIP | 用户未命中 88VIP 人群 | 组件不展示 | — |

**信息层级**（高到低）：
1. 88VIP 标识 + 组件标题（身份教育）
2. 2 个主推权益（核心转化）
3. "+5 项更多权益" 按钮（查全路径）
4. 组件优先级：**高于首页普通权益组件**（PRD 明示）

**交互说明**：
- 大卡点击：跳运营配置的 URL（默认地址 / 自定义地址）
- 按钮点击：弹半屏浮层（浮层从底部上滑动画 300ms）
- 浮层内 7 项权益点击：单独跳对应权益详情（不关闭浮层的情况下仅记录行为）

### 触点 2 · 列表页 · 88VIP 权益组件 + 浮层

**触发条件**：88VIP 用户进入列表页 + 当前列表无"会员快筛组件"（与快筛互斥 · 快筛优先级高）

**新增/改动元素**：

| 元素 | 位置 | 结构 | 内容规则 | 约束 |
|------|------|------|----------|------|
| 入口氛围 | 列表顶部 | 88 样式横条（icon + 文字 + 筛选按钮）| 「88VIP 享 3 项权益 · 筛选」| 文字 ≤ 15 字 |
| 筛选按钮 | 横条右侧 | 胶囊按钮 | 文案「筛选」| 固定 |
| 浮层 | 点击横条弹出半屏 | 权益模块（专享价 / 闪购早餐券 / 旅行消费券）| 标题「88VIP 享 x 项权益」（x 取运营配置数）| 灰字说明「带对应标签酒店可用」 |
| 浮层模块优先级 | 浮层内部 | 权益 > 优惠券 > 挑战任务 | 固定顺序 | — |

**状态枚举**：

| 状态 | 触发条件 | 展示内容 | 交互行为 |
|------|----------|----------|----------|
| 有快筛 | 列表命中会员快筛 | **88VIP 组件不展示**（快筛优先）| — |
| 无快筛 · 88VIP | 正常状态 | 横条 + 筛选按钮展示 | 点击横条弹浮层；点击筛选按钮应用筛选 |
| 浮层展开 | 用户点击横条 | 半屏浮层展示权益明细 | 点击关闭 / 点击外部收起 |
| 筛选激活 | 用户点击筛选后 | 列表召回 88VIP 专享价商品（RP 标 21696071 / rate 标 66560072）| 筛选按钮变激活态 |

**信息层级**：
1. "88VIP 享 X 项权益" 标题（身份+数量锚点）
2. 筛选按钮（功能入口）
3. 浮层内权益明细（内容）
4. 灰字说明"带对应标签酒店可用"（边界告知）

### 触点 3 · 综合筛选浮层

**触发条件**：用户在列表页点击综合筛选

**改动元素**：

| 元素 | 位置 | 结构 | 内容规则 | 约束 |
|------|------|------|----------|------|
| 筛选项 | "优惠促销"模块 | 新增「88VIP 专享价」筛选项 | 替换线上【88VIP 优惠】 | 固定文案 |
| 筛选逻辑 | 后端识别 | 根据 RP 标 21696071 + rate 标 66560072 | 召回 88VIP 专享商品 | — |

**状态枚举**：

### 全局约束

- **身份前缀模板**：所有身份表达统一「[身份] · 内容」格式；`·` 是固定分隔符；禁止用「享/可兑/可享」等动词
- **前端通用组件**：身份+份数变量必须由通用 `<IdentityLabel>` 组件承接，禁止各业务端自由拼接
- **88VIP 品牌色**：全链路统一（待 UI 锁定，建议承接现有 88VIP 红黑系）
- **角标形态统一**：88VIP 角标必须异形（非方形），会员卡片 / 订详页本单可享等复用同一组件
- **文案字数约束**：
  - 标签 ≤ 8 字（合并 icon 含金额可放宽到 20 字）
  - 标题 ≤ 15 字
  - 说明 ≤ 30 字
- **信息密度**：一屏信息块 ≤ 5 个；权益标签同层级 ≤ 3 个，超出折叠
- **金额预估标注**：列表合并 icon 金额字段必须带「约」字 + 浮层免责声明「实际优惠以下单页为准」
- **合规兜底**：涉及金额预估（合并 icon）和"不算价"红包的文案样式，须法务锁定后定版
- **跨端一致性**：飞猪/淘系/美团/携程四端上线的相同触点必须用同一套文案模板；端差异只在 UI 层
