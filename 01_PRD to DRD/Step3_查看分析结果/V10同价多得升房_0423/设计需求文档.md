# V10 同价多得 & 升房 — 设计需求文档（DRD）

> 产出时间：2026-05-11
> 范围：只到 DRD（不含设计稿）
> 设计决策基于：7 次 `ask_user_question` 设计师点选 + `DesignPrinciple.md` + `BussinessExp.md`

---

## 一、核心策略

**一句话方向**：把「权益有没有」升级为「权益用没用上」—— 前台轻（只让用户看到价值与使用方法），后台重（AI 托管完成升房等复杂操作）。

**设计关键词**：主动展示 · 预估锚点 · AI 托管 · 价值-动作一体 · 状态透明

**三条底层逻辑**：
1. **金额不再追求精确** — 模块级「预估」替代单项精确，既保留锚定效应又规避合规风险
2. **主动性 > 确定性** — AI 代用户执行升房，前台只做状态播报
3. **主视野直出** — 核心权益在主流量路径上必露出，不靠用户点开二级

---

## 二、关键决策点（7 条）

### 决策 1 · 权益金额表达方式
- **推荐**：预估锚点 + 点击展开计算口径
- **原因**：`BE 二.V10.确定性偏好` 要求不用模糊表述但不能过精确；`DP 三.锚定效应` 保留价值锚点；`DP 一.2 规则前置` 点击透明化
- **具体方案**：标签文案统一为「预估 ¥XX」（不写"约"、不写精确小数）；点击后在 chat 内出算法说明

### 决策 2 · 下单页权益展示位置
- **推荐**：会员卡主视野直出 3 个高价值权益 + 模块级预估总价值
- **原因**：`BE 一.会员权益.感知弱`（权益页访问率 <5% 铁证）；`DP 一.2 决策信息可见率 100%`
- **具体方案**：卡片主区域 = 3 个权益图标 + 利益点 + 底部「本单权益预估价值 ¥XX」；其他权益折叠为「+N 项权益」

### 决策 3 · 升房交互模式
- **推荐**：订详吸底 AI chat 入口 + AI 全托管
- **原因**：利用已有吸底 chat 组件（零新增入口）；`BE 一.会员权益.自动享受>手动领取` +30%；`BE 二.V10.确定性偏好`
- **具体方案**：用户点吸底 chat → 输入升房需求 → AI 代为致电/提交 → chat 内播报结果；**AI 是执行者不是对话者**

### 决策 4 · 权益使用指引结构
- **推荐**：每项权益 = 名称 + 一句利益点；模块底部 = 预估总价值
- **原因**：避开「每项拆金额算不准」的坑；`DP 一.3 标题含利益点`
- **具体方案**：
  - 单项格式：「`[权益名]` · `[利益点 ≤ 15字]`」（不出具体金额）
  - 模块底部：「本单权益预估价值约 ¥XXX（点击看明细）」

### 决策 5 · 升房入口触达时机
- **推荐**：D6 · 改订详底部吸底 chat 入口默认引导词（不新增下单成功页入口）
- **原因**：不新增入口的情况下，通过引导词主动性转化触达效率
- **具体方案**：吸底 chat 引导词从通用（"有问题问 AI"）改为场景化主动（例：「可以帮你申请免费升房哦」），白名单场景才展示此文案

### 决策 6 · Chat 开场引导文案（已定稿）
- **推荐**：D6 · 文案定稿
- **原因**：`DP 一.3 零歧义` + `BE 二.V10.确定性偏好`（状态+时间）
- **具体方案**：
  > **「收到，飞猪会在你入住前致电酒店，为你申请『客房升级』，最晚会在入住当日 12:00 前答复升房结果。」**

  文案骨架：① 收到确认 ② 服务动作（致电酒店） ③ 明确截止时间（入住当日 12:00 前） ④ 承诺答复

### 决策 7 · 升房异常态信息架构
- **推荐**：D4 · 失败态三段式
- **原因**：`DP 二.P1 关键信息缺失→误解`；`DP 四.模糊状态 P1 反模式`；`BE 二.V10.确定性偏好`
- **具体方案**：失败消息固定三段：
  - 段 1 · 原因（酒店已满房 / 会员等级不符 / 本单非白名单）
  - 段 2 · 下一步建议（换日期 / 换酒店 / 保留常规房）
  - 段 3 · 情绪安抚（一句，避免压力文案）

## 三、文案规范

| 场景 | 推荐文案 | 避免文案 | 原则依据 |
|------|----------|----------|----------|
| 权益模块总价值 | 本单权益预估价值约 ¥XXX | 权益价值 ¥XXX / 立省 ¥XXX | `DP 二.P1 价格偏差≤5%` · `BE 二.V10.确定性偏好` |
| 单项权益表达 | 免费早餐 · 入住当天直接报房号 | 免费早餐(价值¥528) | `DP 一.3 含利益点` · 避开单项金额算不准 |
| 吸底 chat 引导词 | 可以帮你申请免费升房哦 | 有问题随时问我 / 点击咨询 | `DP 一.3 标题含利益点` · `BE 一.会员权益.感知弱` |
| Chat 开场白 | 收到，飞猪会在你入住前致电酒店，为你申请「客房升级」，最晚会在入住当日 12:00 前答复升房结果 | 升房处理中，请耐心等待 | `DP 一.3 零歧义` · `BE 二.V10.确定性偏好` |
| 升房进行中 | 飞猪正在为你联系酒店，预计 XX 前给结果 | 处理中 / 申请中 | `DP 四.模糊状态 P1 反模式` |
| 升房成功 | 已为你升级至 X 房型！入住时出示身份证即可 | 恭喜升级！请等待通知 | `BE 二.V10.庆祝感` · 附带下一步动作 |
| 升房失败 | 酒店已满房，可为你换到 X 日期或保留当前房型 | 申请失败 / 未能升级 | 三段式，必须含下一步 |
| 计算口径说明（金额点击后） | 权益预估价值按「参考酒店同类房型单早/房价差」预估，实际以酒店现场为准 | — | `DP 一.2 规则前置` |

**文案原则**：
- 所有金额必须是"预估"，禁止精确到元（如 ¥187）或声称"立省"
- 权益必须「名称 + 利益点」双表达，单独一个名词（如"免费早餐"）视为未完成
- 失败/异常文案必须含"下一步动作"，禁止只交代原因
- 禁用损失框架（"错过就没了"）和虚假紧迫（"最后 X 名额"）

## 四、风险提示

| 风险 | 场景 | 规避方案 |
|------|------|----------|
| 预估金额仍被用户当精确值维权 | 权益按预估¥500 算，实际只值 ¥300 | chat 内计算口径说明必须含「以酒店现场为准」免责声明；法务锁定文案 |
| AI 升房托管失败率高导致期望落空 | 用户看到「正在申请」却很多未成功 | 开场白明确"是申请不是保证"；成功文案才说升级 |
| 非白名单场景用户困惑 | 用户看权益介绍但发现本单不能用（本期不做兜底） | 会员卡片在非白名单下隐藏/降级展示（本期交给前端判断，UI 不做特殊兜底） |
| 吸底 chat 引导词破坏页面一致性 | 非升房场景下引导词仍指向升房 | 引导词文案需基于白名单+会员等级双条件动态切换 |
| 模块级预估总价值与单项利益点不对应 | 用户质疑"怎么算出来¥XXX" | 点击总价值入 chat，列出本单适用的权益清单 + 参考价来源 |

## 五、优先级排序

| 优先级 | 设计任务 | 理由 |
|--------|----------|------|
| P0 | 下单页会员卡片重构（决策 2+4） | 权益价值感知的主战场，`BE 一.会员权益.感知弱` 的核心解决方案 |
| P0 | Chat 开场白 + 升房进行中/成功/失败状态文案（决策 6+7） | AI 托管的用户信任基石，文案定稿即可开发 |
| P1 | 吸底 chat 引导词动态切换（决策 5） | 触达入口，轻量改动 0.5h 但效果显著 |
| P1 | 金额点击展开计算口径交互（决策 1） | 合规防线，必须在上线前完成 |
| P2 | AI 全托管升房后端逻辑 | 开发侧为主，设计侧只出状态枚举 |
| **scope 外** | 会员等级视觉强化 / 非白名单兜底 | 本期不做（机会点 D+E） |

## 六、设计规格

### 触点 1 · 下单页 · 会员权益卡片

**触发条件**：
- 用户下单页展示会员卡片时
- 且该订单酒店在白名单内
- 且用户有 V10 会员等级

**新增/改动元素**：

| 元素 | 位置 | 结构 | 内容规则 | 约束 |
|------|------|------|----------|------|
| 权益主区域 | 卡片中部 | 3 个权益图标 + 名称 + 利益点 | 按预估价值 Top3 排序 | 最多 3 个，超出折叠 |
| 权益折叠入口 | 权益主区域右下 | 「+N 项权益」可点击 | N = 总权益数 - 3 | 必现 |
| 模块级总价值 | 卡片底部独立一行 | 「本单权益预估价值约 ¥XXX」+ 可点击图标 | 金额取所有权益预估价值汇总（整数） | 字数 ≤ 20 |
| 计算口径入口 | 总价值行右侧 ⓘ 图标 | 点击入 chat | — | — |

**状态枚举**：

| 状态 | 触发条件 | 展示内容 | 交互行为 |
|------|----------|----------|----------|
| 默认态 | 白名单 + V10 会员 | 3 权益 + 总价值 + 计算入口 | 点击 ⓘ 入 chat；点击折叠入口展开全部权益 |
| 非白名单 | 酒店不在白名单 | 整个会员权益卡片隐藏 | — |
| 无 V10 | 用户未达 V10 | 整个会员权益卡片隐藏 | — |

**信息层级**（从高到低）：
1. 3 个权益的利益点文案（用户决策核心）
2. 权益图标（视觉引导）
3. 「本单权益预估价值约 ¥XXX」（模块锚定）
4. 「+N 项权益」折叠入口
5. ⓘ 计算口径入口

**交互说明**：
- 点击 ⓘ：入 chat，chat 内自动发出系统消息「权益预估价值按...计算，以酒店现场为准」+ 本单适用权益清单
- 点击「+N 项权益」：卡片内展开所有权益列表（不跳转）
- 点击单个权益图标：无交互（仅视觉元素）

### 触点 2 · 订单详情页 · 底部吸底 Chat 入口

**触发条件**：
- 订单状态为待入住
- 酒店在白名单内
- 用户有 V10 会员等级

**新增/改动元素**：

| 元素 | 位置 | 结构 | 内容规则 | 约束 |
|------|------|------|----------|------|
| 吸底 chat 入口 | 订详底部吸底（已存在） | 图标 + 引导词 | 引导词改为场景化文案 | 字数 ≤ 12 |

**状态枚举**：

| 状态 | 触发条件 | 展示内容 | 交互行为 |
|------|----------|----------|----------|
| 默认态（触发条件满足） | 白名单 + V10 + 待入住 | 「可以帮你申请免费升房哦」 | 点击入 chat |
| 已发起升房 | 用户已点击入口发起过升房 | 「查看升房结果」 | 点击入 chat |
| 非触发 | 不满足触发条件 | 保持通用引导词（原逻辑） | 点击入 chat |

**信息层级**：同吸底组件原有层级，不改变视觉权重

**交互说明**：
- 点击行为：打开 chat 面板，chat 自动发出开场白（见触点 3 · 开场状态）
- 非触发条件下引导词保持原通用文案，由前端根据条件动态切换

---

### 触点 3 · Chat 对话 · 升房场景

**触发条件**：
- 用户从触点 2（吸底 chat 入口）进入 chat
- 且处于升房可触发条件下

**新增/改动元素**：

| 元素 | 位置 | 结构 | 内容规则 | 约束 |
|------|------|------|----------|------|
| 系统开场消息 | Chat 首条 AI 消息 | 确认 + 服务动作 + 截止时间 | 定稿文案 | 法务锁定 |
| 状态消息 | Chat 后续 AI 消息 | 状态文字 | 按状态枚举 | 每条 ≤ 40 字 |
| 失败三段消息 | 失败态 AI 消息 | 三段分段 | 原因 + 建议 + 安抚 | 整体 ≤ 120 字 |

**状态枚举**（全部对应 chat 消息气泡）：

| 状态 | 触发条件 | 展示内容 | 交互行为 |
|------|----------|----------|----------|
| 开场 | 用户进入 chat | 「收到，飞猪会在你入住前致电酒店，为你申请『客房升级』，最晚会在入住当日 12:00 前答复升房结果。」 | 无操作，等待用户确认 |
| 进行中 | 系统已提交升房流程 | 「飞猪正在为你联系酒店，预计 XX 前给结果」 | 订详页顶部状态同步为「升房处理中」 |
| 成功 | 酒店确认升房 | 「已为你升级至 `[房型]`！入住时出示身份证即可，无需其他操作。」 | 订详页状态同步为「已升级」 |
| 失败 · 已满房 | 酒店反馈满房 | 段 1：「酒店当日已满房，暂时没有升级空间」<br>段 2：「你可以换到 X 日期继续升级，或保留当前房型」<br>段 3：「本单其他权益不受影响，仍为你保留」 | 附「换日期」「保留当前房」二选一按钮 |
| 失败 · 会员不符 | 系统判断会员等级下降 | 段 1：「当前会员等级暂不符合升级条件」<br>段 2：「你可以查看如何升级到 F4 等级」<br>段 3：「其他权益仍可正常使用」 | 附「如何升级」按钮 |
| 失败 · 非白名单 | 提示本单非白名单（兜底） | 段 1：「本酒店暂不支持免费升房」<br>段 2：「你可以查看支持的酒店列表」<br>段 3：「本单其他权益不受影响」 | 附「查看支持酒店」按钮 |
| 权益计算口径（被动触发） | 用户点触点 1 的 ⓘ | 「权益预估价值按『参考酒店同类房型单早/房价差』预估，实际以酒店现场为准。本单为你准备了：权益 1 预估 ¥X / 权益 2 预估 ¥X / ...」 | 无按钮 |

**信息层级**：
1. 开场白（建立信任）
2. 进行中/结果（核心信息）
3. 行动按钮（下一步）
4. 安抚文字（情绪）

**交互说明**：
- 所有 AI 消息支持复制长按
- 失败态附按钮点击后跳转对应页面，不再在 chat 内继续对话
- 升房状态变更时，chat 和订详页顶部状态必须同步（不得一方更新另一方不更新）

---

### 全局约束

- 权益展示最多 **3 个**，超出折叠为「+N 项权益」
- 单项利益点文案最多 **15 字**
- 权益名称最多 **8 字**
- 模块级预估总价值文案最多 **20 字**
- 吸底 chat 引导词最多 **12 字**
- Chat 开场白为法务锁定文案，Agent 不得改写
- 所有金额必须带「预估」前缀，禁止出现精确金额和「立省」「节省」等框架
- 全链路一致性：chat 状态变更与订详页顶部状态**必须同步**；白名单/会员等级判断逻辑三端一致
