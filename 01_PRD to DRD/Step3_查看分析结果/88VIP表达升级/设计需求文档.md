## 一、核心策略

把「88VIP 身份」从后台跑的优惠数字，变成**全链路可见、可算、可复购**的一致视觉语言。

**设计关键词**：立省具体金额 · 差异对比 · 金色异形 · 一次性收起 · 庆祝闭环

**整体设计方向**（1 句话）：
所有 88 表达统一为「**88VIP 立省 ¥XX（普通用户 ¥XX）** + 金色渐变 44° 异形」；首屏上滑一次全收让位房型；下单/订详把省了多少做成庆祝闭环（新增）。

## 二、关键决策点

### 决策 1 · 88 表达口径：**文案承载实惠数字，视觉承载身份感**
- **推荐口径**：全链路统一「88VIP 立省 ¥XX（普通用户 ¥XX）」
- **原因**：`BE 一.会员·权益感知弱` + 画像 A「还不如直接给我个会员折扣」+ 画像 B「跟普通用户没两样」→ 文案必须有数字，身份感交给视觉
- **具体方案**：F1/F2/F3/F5/F6/F7/F8 所有触点文案主体 = 立省金额，身份感由金色渐变 44° 异形 + 88VIP logo 承载；**禁止出现「88VIP 专享优惠」等无数字的纯身份叙事**

### 决策 2 · F3 列表 icon：**宽横条独占一行 + 点击浮层逐项**
- **推荐**：宽横条 icon（方案 A）
- **原因**：`DP 一.3 文案可读·标题含利益点数字` + `BE 二.V10·差异感知`
- **具体方案**：卡片内独占一行金色渐变异形条「立省 ¥XX · 88 专享」+ 右侧小箭头；点击从 icon 位置向下滑出浮层列明「88 专享价 -¥A / 旅行消费券 -¥B / 首单立减 -¥C」

### 决策 3 · 早餐差异对比句式：**全链路统一 + 数据缺失兜底**
- **推荐**：全链路统一（方案 A）
- **原因**：`DP 三.差异感知` + 1.E 张力 3/8 已拍板「全局差异对比」
- **具体方案**：F5/F6/F8 主句「88VIP 专享免早（普通用户 ¥XX/份）」；详情营销标空间紧缩写「免早 · 普通 ¥XX」；商家普价数据缺失时回落「免早 · 省 ¥XX」（不回退单向）

### 决策 4 · F7 下单页金额：**两层结构**
- **推荐**：首屏实付 + 立省庆祝（方案 A）
- **原因**：`DP 一.1 链路连贯` + `DP 三.认知负荷·下单一屏 ≤5 信息块` + `BE 二.V10 庆祝感`
- **具体方案**：首屏「实付 ¥XX · 88VIP 立省 ¥YY」；点击展开明细「专享价 -¥A / 消费券 -¥B / 来自【商家】首单立减 ¥C」；**注册红包不入总价计算**（张力 5 方向一 + Q5 文案修正汇合）

### 决策 5 · 首屏 88 表达上滑收起：**一次性全收 + 下滑恢复**
- **推荐**：一次性全收（方案 A）
- **原因**：`DP 一.4 主操作唯一突出` + 1.E 张力 4 已拍板「全部收起含 F3 icon」
- **具体方案**：列表页监听滚动，上滑 ≥120px 触发 300ms 淡出，F1 权益组件/F2 权益入口/F3 列表 icon/F5 营销标同时隐藏；下滑任意距离即刻全体淡入

### 决策 6 · F2 列表页「只看 88 房」筛选：**快筛栏首位金色异形按钮**
- **推荐**：独立快筛按钮（方案 A）
- **原因**：`DP 一.1 ≤3 步` + 1.E 张力 6 已拍板「筛选项优先级高于默认排序置顶」
- **具体方案**：列表顶部快筛栏首位加金色异形按钮「只看 88 房」，未开启默认灰色 outline；点击激活后金色异形与决策 2 视觉规范对齐；随 F1/F2 权益栏一起上滑收起

### 决策 7 · F1 首页权益组件：**3 格横滑卡片跳已有业务页**
- **推荐**：3 格横滑（方案 A）
- **原因**：`DP 一.1 链路连贯` + Q2 已拍「承接页本期不做，只设计入口 + 预留 URL」
- **具体方案**：每格一个利益点带场景数字（¥1 起闪购早餐 / 免费升级万豪白金 / X 家酒店可用权益云）；点击跳已有闪购列表/权益中心/搜索结果页；**不新建承接页**

### 决策 8 · 身份视觉载体统一规范：**金色渐变 44° 异形**
- **推荐**：金色渐变（方案 A）
- **原因**：`DP 三.视觉层次` + 画像 B 尊贵感 + 业内通识视觉
- **具体方案**：统一 token = 金色渐变底 `#E8B855 → #D4A243` + 44° 斜切异形 + 左侧 88VIP logo 16px + 右侧数字/文字主体；浅背景（订详/下单）加 2px 白描边弥补对比度

### 决策 9 · ⚠️ 新增 · 权益使用后反馈闭环：**订详/支付成功页顶部庆祝条**
- **推荐**：庆祝条（方案 A）
- **原因**：2.A 业务目标审核发现「88VIP 留存复购」⚠️ 缺方案 + `BE 二.V10 庆祝感` + 画像 A「权益感觉假」原语
- **具体方案**：支付成功 + 订详页顶部金色庆祝条「本次 88VIP 为你节省 ¥XX · 已累计省 ¥YY」；点击跳「我的 88VIP 账单」；累计数据不齐时兼容「本月省 ¥XX」

| F7 下单页明细展开 | 「来自【商家】首单立减 ¥C」 | 「享【商家】注册红包」 | Q5 文案修正 |

**文案原则**：
- **P1 · 数字必须上首屏**：所有 88 表达首屏可见处必含具体金额（¥XX / ¥XX/份），不把数字收进浮层
- **P2 · 对比必须成对**：出现「88VIP 专享 X」必须配对「普通用户 Y」（数据缺失走兜底「省 ¥XX」）
- **P3 · 禁用压力文案**：不出现「不领就亏」「错过就没」；用庆祝感替代（`BE 二.V10` 已定）
- **P4 · 禁用纯身份叙事**：所有触点不得出现「88VIP 专享优惠」类无数字短句
- **P5 · 商家名称必须拼入首单立减**：「来自【商家名】首单立减 ¥XX」（Q5）

## 四、风险提示

| 风险 | 场景 | 规避方案 |
|---|---|---|
| 商家普通用户价数据缺失 | F5/F6/F8 差异对比句式无数据源 | 兜底文案「免早 · 省 ¥XX」；DRD 交付时要求后端同步补齐商家普价字段 |
| 浅背景页金色对比度不足 | F7 下单页 / F8 订详页（通常白底）| 视觉规范强制加 2px 白描边；设计稿交付需出浅/深背景双版 |
| 上滑收起后 88 房型无法识别 | 用户在列表页快速滑动时 88 身份感丢失 | 用「只看 88 房」快筛按钮弥补（决策 6）；筛选激活后用户主动筛 88 房不依赖 icon |
| F1 权益组件利益点选择运营权 | 3 格内容非设计决定 | DRD 只定卡片结构和视觉规范，具体利益点由运营配置；但每格必须遵守「场景数字 + 动作」文案规范 |
| 累计省金额数据未打通 | F8 庆祝条「已累计省 ¥YY」可能取不到 | 兜底「本月省 ¥XX」；若本月也取不到，降级为只显示「本次 88VIP 为你节省 ¥XX」 |
| 「88VIP 可兑免早」改为对比句式引起商家投诉 | 商家可能认为「被砍价」 | 文案由飞猪统一承担，不修改商家协议价；DRD 附风险提示表给到 PM 同步 BD |
| Q6 退改确定性本期延后 | 画像 C 原语「退改不确定不敢下单」未回应 | 记入 Backlog 下版本「88VIP 退改无忧/售后优先」；本期在支付成功庆祝条处为未来承诺留视觉位 |
| 双识别口径（场景码 / is88Vip）并存 | 工程后续新增 88 券需改两处 | 设计侧不介入（1.E 张力 7 已脱约）；DRD 末尾注明工程侧跟进项 |

## 六、设计规格

### 页面 / 触点 1 · F3 列表页 88 优惠 icon

**触发条件**：房型是 88VIP 专享价房型（后端返回 `is88Vip=true`）

**新增 / 改动元素**：

| 元素 | 位置 | 结构 | 内容规则 | 约束 |
|---|---|---|---|---|
| 88 异形横条 | 房型卡片内、价格行下方 | 金色渐变底 + 88VIP logo(16px) + 文字主体 + 右箭头(12px) | 「立省 ¥XX · 88 专享」（XX 取本单可享最大优惠金额）| 高度 28px · 字号 13sp · 异形斜切 44° · 左内边距 12px · 右箭头距边 8px |
| 优惠明细浮层 | 从 icon 向下滑出 / 点击展开 | 纵向列表，每项 = 名称 + 金额 | 「88 专享价 -¥A」「旅行消费券 -¥B」「来自【商家】首单立减 -¥C」 | 浮层最大高度 4 行（≤4 项），超出内滚；300ms 展开/收起动画 |

**状态枚举**：

| 状态 | 触发条件 | 展示内容 | 交互行为 |
|---|---|---|---|
| 默认态 | 88 房型首次进入视口 | 异形横条展示「立省 ¥XX · 88 专享」 | 点击整条可展开浮层 |
| 浮层展开态 | 用户点击异形横条 | 浮层从 icon 位置滑出 | 点击外部 / 再次点击横条收起 |
| 数据缺失态 | 最大立省金额取不到 | 退化为「88VIP 专享 · 省钱立现」（极端兜底，禁出现）| 无浮层 · 点击无响应 |
| 滚动收起态 | 决策 5 触发（上滑 ≥120px）| 横条整体淡出，高度归零 | 下滑任意距离即刻恢复 |

**信息层级**（从高到低）：
1. 立省金额数字（最大字号 · 品牌色）
2. 「88 专享」身份词（加粗 · 次字号）
3. 88VIP logo + 箭头（辅助视觉）

**交互说明**：
- 点击行为：整条 tap → 浮层展开 · 浮层内部点击 → 跳单间详情页
- 滑动 / 展开：浮层 300ms ease-out 展开，收起 200ms ease-in
- 动效要求：卡片滚入视口时 icon 渐显（200ms）避免闪烁

### 全局约束

- **88 视觉 token**：金色渐变 `#E8B855 → #D4A243` · 44° 异形斜切 · 88VIP logo 16px · 浅背景加 2px 白描边
- **权益展示上限**：列表页每卡片 88 相关元素 = 1（横条 icon 一条）· 首屏 88 表达层数 ≤ 4（F1/F2/F3/F5）
- **数字规则**：所有立省金额必须显示整数人民币（¥XX），不显示小数；金额 ≥ ¥1000 用千分位
- **文案长度**：异形标 ≤ 14 字 · 庆祝条主文案 ≤ 26 字 · 首屏立省行 ≤ 18 字
- **信息密度**：下单页金额模块首屏只显 2 行（实付 + 立省）· 详情页一屏 ≤ 5 信息块
- **全链路一致性**：所有 88 表达主文案 = 「88VIP 立省 ¥XX」口径 · 所有对比句式 = 「（普通用户 ¥XX）」口径 · 所有身份视觉 = 金色渐变 44° 异形
- **收起规则**：上滑 ≥120px 触发所有 88 表达 300ms 淡出 · 下滑任意距离立即恢复
- **差异对比兜底规则**：商家未配普通用户价时回落「省 ¥XX」而非回退单向
- **首单立减规则**：独立展示于总价下方 · 不入实付金额计算 · 文案「来自【商家】首单立减 ¥XX」
